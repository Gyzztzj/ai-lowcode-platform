## 1. 项目概述

AI 低代码平台是一个基于 React 的可视化 AI 工作流构建平台，允许用户通过拖拽方式创建复杂的 AI 应用，包括聊天机器人、知识库检索、工作流编排等功能。

### 1.1 核心特性
- 可视化工作流编辑器（React Flow）
- 多节点类型支持（开始、结束、用户输入、系统提示、LLM、知识库、条件、变量设置）
- 知识库管理与文档上传
- 对话历史记录
- 模型配置管理
- API 密钥管理
- 应用分享功能

### 1.2 技术选型原则
- 类型安全优先（TypeScript）
- 开发体验优先（Vite）
- 生态成熟（React 生态）
- 性能优先（轻量级状态管理）

### 3.2 分层说明

#### 表现层 (Presentation Layer)
- **页面组件**：路由对应的页面，位于 `pages/`
- **布局组件**：页面布局相关，位于 `components/layout/`
- **UI 组件**：基础 UI 组件（shadcn/ui），位于 `components/ui/`
- **业务组件**：业务相关组件，位于 `components/` 各子目录

#### 业务层 (Business Layer)
- **自定义 Hooks**：业务逻辑复用，位于 `hooks/`
- **Store**：全局状态管理，位于 `store/`
- **服务**：业务服务封装，位于 `services/`

#### 数据层 (Data Layer)
- **API Client**：HTTP 请求封装，位于 `lib/api-client.ts`
- **TanStack Query**：数据缓存与同步，全局配置于 `App.tsx`
- **localStorage**：持久化存储（如 Token）

## 5. 核心功能模块

### 5.1 工作流编辑器

工作流编辑器是平台的核心功能，使用 React Flow 实现可视化拖拽编排。

#### 5.1.1 节点类型

| 节点类型 | 说明 | 文件位置 |
|---------|------|---------|
| Start | 开始节点，工作流起点 | `components/builder/nodes/StartNode.tsx` |
| End | 结束节点，工作流终点 | `components/builder/nodes/EndNode.tsx` |
| UserInput | 用户输入节点，获取用户输入 | `components/builder/nodes/UserInputNode.tsx` |
| SystemPrompt | 系统提示词节点，设置 LLM 系统提示 | `components/builder/nodes/SystemPromptNode.tsx` |
| Llm | LLM 调用节点，调用大语言模型 | `components/builder/nodes/LlmNode.tsx` |
| KnowledgeBase | 知识库检索节点，检索相关文档 | `components/builder/nodes/KnowledgeBaseNode.tsx` |
| Condition | 条件判断节点，支持分支逻辑 | `components/builder/nodes/ConditionNode.tsx` |
| VariableSet | 变量设置节点，设置或修改变量 | `components/builder/nodes/VariableSetNode.tsx` |

#### 5.1.2 节点基类设计

所有自定义节点继承自 `BaseNode`，提供统一的样式和功能。

```typescript
// components/builder/nodes/BaseNode.tsx

  data: NodeData;
  selected: boolean;
  type: string;
  icon: React.ReactNode;
  children?: React.ReactNode;
}

function BaseNode({ data, selected, type, icon, children }: BaseNodeProps) {
  return (
    <div className={cn(
      "min-w-[180px] bg-white border-2 rounded-lg shadow-md transition-all",
      selected ? "border-primary ring-2 ring-primary/20" : "border-gray-200"

      {/* 节点头部 */}
      <Handle type="target" position={Position.Top} />
      <div className="px-4 py-2 border-b bg-gray-50 rounded-t-lg flex items-center gap-2">
        {icon}
        <span className="font-medium text-sm">{data.label || type}</span>
      </div>
      {/* 节点内容 */}
      {children}
      <Handle type="source" position={Position.Bottom} />
    </div>
  );
}
```

## 6. 状态管理方案

### 6.1 Zustand Store 设计

项目使用 Zustand 进行全局状态管理，分为三个主要 Store。

### 6.2 User Store - 用户状态管理

`store/userStore.ts` - 管理用户认证状态、用户信息、Token 等。

```typescript
import { create } from "zustand";
import type { User } from "@/types/index";
import { authApi } from "@/lib/api-client";

interface UserState {
  user: User | null;
  token: string | null;
  refreshToken: string | null;
  isLoading: boolean;
  logout: () => Promise<void>;
  fetchCurrentUser: () => Promise<void>;
  setAuth: (user: User, accessToken: string, refreshTokenVal?: string) => void;
  setToken: (token: string) => void;
}

export const useUserStore = create<UserState>((set, get) => ({
  user: null,
  token: localStorage.getItem("token"),
  refreshToken: localStorage.getItem("refreshToken"),
  isLoading: false,

  logout: async () => {
    try {
      const { refreshToken } = get();
      if (refreshToken) {
        await authApi.logout({ refreshToken });
      }
    } catch (error) {
      console.error("userStore: 调用后端 logout API 失败:", error);
    }
    
    localStorage.clear();
    sessionStorage.clear();
    
    set({ user: null, token: null, refreshToken: null, isLoading: false });
  },

  fetchCurrentUser: async () => {
    try {
      set({ isLoading: true });
      const response = await authApi.getCurrentUser();
      set({ user: response.user, isLoading: false });
    } catch (error) {
      console.error("userStore: fetchCurrentUser 失败:", error);
      localStorage.removeItem("token");
      localStorage.removeItem("refreshToken");
      set({ user: null, token: null, refreshToken: null, isLoading: false });
    }
  },
  
  setAuth: (user, accessToken, refreshTokenVal) => {
    localStorage.setItem("token", accessToken);
    if (refreshTokenVal) {
      localStorage.setItem("refreshToken", refreshTokenVal);
    }
    set({ user, token: accessToken, refreshToken: refreshTokenVal || null, isLoading: false });
  },

  setToken: (token) => {
    localStorage.setItem("token", token);
    set({ token });
  },
}));
```

### 6.3 App Store - 应用状态管理

`store/appStore.ts` - 管理应用列表、知识库、模型等全局数据。

### 6.4 Builder Store - 编辑器状态管理

`store/builderStore.ts` - 管理工作流编辑器的节点、边、选中状态等。

### 6.5 状态管理最佳实践

1. **按需订阅**：使用选择器函数订阅所需状态，避免不必要的重渲染
2. **操作原子化**：每个 Action 只做一件事
3. **异步操作**：在 Store 中处理异步请求，组件只调用方法
4. **持久化**：使用 `zustand-persist` 或直接操作 localStorage 持久化关键状态

## 7. 路由配置

### 7.1 路由结构

使用 React Router v7 配置路由，支持嵌套路由和路由守卫。

```typescript
// AppRoutes.tsx
import { Navigate, Route, Routes } from "react-router-dom";
import ProtectedRoute from "./components/ProtectedRoute";
import MainLayout from "./components/layout/MainLayout";

const AppRoutes = () => {
  return (
    <Routes>
      {/* 公开路由 */}
      <Route path="/login" element={<Login />} />
      <Route path="/register" element={<Register />} />
      <Route path="/share/:shareId" element={<Share />} />

      {/* 需要认证的路由 */}
      <Route element={<ProtectedRoute />}>
        <Route element={<MainLayout />}>
          <Route path="/" element={<Navigate to="/chat" replace />} />
          <Route path="/chat" element={<Chat />} />
          <Route path="/apps" element={<Apps />} />
          <Route path="/apps/:appId/builder" element={<Builder />} />
          <Route path="/models" element={<ModelManagement />} />
          <Route path="/knowledge" element={<Knowledge />} />
          <Route path="/knowledge/:id" element={<KnowledgeDetail />} />
        </Route>
      </Route>

      {/* 404 重定向 */}
      <Route path="*" element={<Navigate to="/" replace />} />
    </Routes>
  );
};
```

### 7.2 路由守卫

`ProtectedRoute` 组件检查用户是否已登录，未登录则重定向到登录页。

```typescript
// components/ProtectedRoute.tsx
import { Navigate, Outlet, useLocation } from "react-router-dom";
import { useUserStore } from "@/store/userStore";

function ProtectedRoute() {
  const location = useLocation();
  const token = useUserStore((state) => state.token);

  if (!token) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return <Outlet />;
}
```

### 7.3 路由懒加载

对于大型页面组件，使用 React.lazy 和 Suspense 实现路由级别的代码分割。

```typescript
import { lazy, Suspense } from "react";

const Chat = lazy(() => import("./pages/Chat"));
const Apps = lazy(() => import("./pages/Apps"));

// 在路由中使用
<Suspense fallback={<PageLoading />}>
  <Route path="/chat" element={<Chat />} />
</Suspense>
```

## 8. API 交互

### 8.1 Axios 配置

`lib/axios.ts` 配置 Axios 实例，包含请求拦截器和响应拦截器。

```typescript
import axios from "axios";
import { useUserStore } from "@/store/userStore";

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_URL || "http://localhost:3000/api",
  timeout: 30000,
  headers: {
    "Content-Type": "application/json",
  },
});

// 请求拦截器：自动添加 Authorization header
apiClient.interceptors.request.use(
  (config) => {
    const token = useUserStore.getState().token;
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器：统一处理响应和错误
apiClient.interceptors.response.use(
  (response) => {
    return response.data;
  },
  async (error) => {
    if (error.response?.status === 401) {
      // Token 过期，清除认证信息
      useUserStore.getState().logout();
    }
    return Promise.reject(error);
  }
);

export default apiClient;
```

```typescript
export const appsApi = {
  list: () => apiClient.get("/apps"),
  get: (id: string) => apiClient.get(`/apps/${id}`),
  create: (data: any) => apiClient.post("/apps", data),
  update: (id: string, data: any) => apiClient.put(`/apps/${id}`, data),
  delete: (id: string) => apiClient.delete(`/apps/${id}`),
};

export const knowledgeApi = {
  list: () => apiClient.get("/knowledge"),
  get: (id: string) => apiClient.get(`/knowledge/${id}`),
  create: (data: any) => apiClient.post("/knowledge", data),
  uploadDocument: (id: string, file: File) => {
    const formData = new FormData();
    formData.append("file", file);
    return apiClient.post(`/knowledge/${id}/documents`, formData, {
      headers: { "Content-Type": "multipart/form-data" },
    });
  },
};
```

```typescript
const queryClient = new QueryClient({

function App() {
```

### 8.4 使用 Query Hooks

在组件中使用 useQuery 和 useMutation。

```typescript
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { appsApi } from "@/lib/api-client";
import { toast } from "sonner";

function AppsPage() {
  const queryClient = useQueryClient();

  const { data, isLoading, error } = useQuery({
    queryKey: ["apps"],
    queryFn: appsApi.list,
  });

  const deleteMutation = useMutation({
    mutationFn: appsApi.delete,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["apps"] });
      toast.success("删除成功");
    },
    onError: () => {
      toast.error("删除失败");
    },
  });

  if (isLoading) return <Skeleton />;
  if (error) return <div>加载失败</div>;

  return (
    <div>
      {data?.apps.map((app) => (
        <AppCard key={app.id} app={app} onDelete={deleteMutation.mutate} />
      ))}
    </div>
  );
}
```

```typescript
// components/xxx/ComponentName.tsx
import { useState, useEffect } from "react";
```
