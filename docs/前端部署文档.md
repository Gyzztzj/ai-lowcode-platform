# 前端部署到 Vercel - 超详细手把手教程

## 准备工作

1. 一个 GitHub 账号
2. 你的代码已经推送到 GitHub 上的 `ai-lowcode-platform` 仓库
3. 一个 Vercel 账号（用 GitHub 登录就行）
4. **服务端已经部署好了，并且有了服务端的地址**（比如：`https://xxx.railway.app`）

---

## 开始部署（每一步都跟着做）

---

### 第 1 步：打开 Vercel 并登录

1. 打开浏览器，访问：https://vercel.com
2. 点击右上角的 **Log In** 或 **Sign Up** 按钮
3. 选择 **Continue with GitHub**
4. 授权 Vercel 访问你的 GitHub 账号（点同意/Authorize）
5. 登录成功后，你会看到 Vercel 的控制台（Dashboard）

---

### 第 2 步：导入项目

1. 在 Vercel 控制台，点击右上角的 **Add New** → **Project**
2. 你会看到你的 GitHub 仓库列表
3. 找到 `ai-lowcode-platform` 仓库，点击旁边的 **Import** 按钮

---

### 第 3 步：配置项目（关键！）

现在你会进入 "Configure Project" 页面，仔细配置：

#### 3.1 Project Name（项目名称）

- 可以随便填，比如：`ai-lowcode-platform`
- 或者保持默认

#### 3.2 Root Directory（根目录，最重要！）

- 点击 **Edit** 或 **Configure** 按钮（在 Root Directory 旁边）
- 输入：`web`（因为前端代码在 web 文件夹里！）
- 点击 **Continue** 或确认

#### 3.3 Framework Preset（框架预设）

- Vercel 会自动检测到这是一个 Vite 项目
- 保持默认即可，不用改

#### 3.4 Build and Output Settings（构建设置）

- **Build Command**: 留空或输入 `pnpm build`
- **Output Directory**: 留空或输入 `dist`

---

### 第 4 步：添加环境变量（最重要！）

在 "Environment Variables" 部分：

1. 点击 **Add** 或 **New** 按钮
2. 填写：
   - **Name**: `VITE_API_URL`
   - **Value**: 你的服务端地址！比如：`https://your-railway-api.railway.app`
   - 注意：地址要加 `https://`，不要有多余的空格
3. 点击 **Save** 或 **Add** 按钮保存

---

### 第 5 步：添加 vercel.json 配置文件（可选但建议，解决刷新 404 问题）

在部署之前，我们先在本地添加一个配置文件（防止刷新页面出现 404）：

1. 在你的项目根目录（不是 web 目录，是最外层！）创建一个新文件，叫 `vercel.json`
2. 内容填写：

```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

3. 保存文件
4. 提交并推送到 GitHub：

```bash
git add vercel.json
git commit -m "add vercel.json for routing"
git push
```

---

### 第 6 步：终于可以部署了！

1. 检查一下所有配置：
   - Root Directory 是 `web` ✓
   - 环境变量 `VITE_API_URL` 已经添加 ✓
   - 服务端地址填对了 ✓

2. 点击 **Deploy** 按钮
3. 等待 1-3 分钟（期间可以看构建日志）
4. 部署成功后，Vercel 会给你一个网址，类似：`https://ai-lowcode-platform.vercel.app`

---

### 第 7 步：验证部署成功

1. 复制 Vercel 分配给你的网址
2. 在浏览器里打开它
3. 如果看到登录页面或首页，说明前端部署成功了！
4. 试试注册/登录，看能不能正常调用 API

---

## 常见问题（按可能性排序）

### Q1: 构建失败了怎么办？

A:

1. 在 Vercel 项目页面，点击 **Deployments** 标签
2. 点击失败的那次部署，看日志
3. 常见原因：
   - 忘记设置 Root Directory 为 `web`
   - 本地先试一下：`cd web && pnpm install && pnpm build`，看本地能不能成功

### Q2: 页面可以打开，但所有 API 请求都失败？

A:

1. 检查环境变量 `VITE_API_URL` 有没有填对
2. 检查服务端的 `ALLOWED_ORIGINS` 有没有包含你的 Vercel 域名
3. 按 F12 打开浏览器开发者工具，看 Console 和 Network 里的错误信息

### Q3: 进入某个页面后，一刷新就 404？

A:

1. 检查第 5 步做了没
2. 确认 `vercel.json` 文件在项目根目录，并且已经推送到 GitHub
3. 重新部署一次

### Q4: 修改了环境变量，但没生效？

A:

1. 修改环境变量后，需要重新部署才能生效
2. 在 Vercel 控制台的项目 **Settings** → **Environment Variables** 里修改，不要在代码里硬编码
3. 修改后，到 **Deployments** 页面，点击最新部署右边的三个点，选择 **Redeploy**

### Q5: Root Directory 找不到在哪里改？

A:

1. 在 Vercel 项目页面，点击 **Settings** 标签
2. 找到 **General** 或 **Build & Development Settings** 部分
3. 找到 **Root Directory** 进行修改

## 本地测试（部署前先试试，避免浪费时间）

如果你想在本地先测试一下前端能不能跑：

```bash
# 进入 web 目录
cd web

# 安装依赖
pnpm install

# 构建
pnpm build

# 预览生产构建
pnpm preview
```

---

## 部署成功后要做的

1. 把 Vercel 分配给你的网址发给你的朋友们
2. 测试一下完整功能（登录、创建应用、知识库等）
3. 如果需要自定义域名，可以在 Vercel 项目 **Settings** → **Domains** 里配置

---

## 修改了代码后怎么更新？

Vercel 默认自动部署：

- 只要你把代码推送到 GitHub 的 main 分支，Vercel 就会自动重新部署
- 你可以在 Vercel 的 **Deployments** 页面看到部署进度
