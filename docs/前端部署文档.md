## 1. 部署前准备

### 1.1 前置条件

- ✅ 已注册 GitHub 账号，代码已推送到 GitHub 仓库
- ✅ 已注册 Vercel 账号（推荐使用 GitHub 账号登录）
- ✅ 后端服务已部署（见服务端部署文档）
- ✅ 本地开发环境可以正常构建

### 1.2 项目检查清单

在部署前，请检查以下内容：

- [ ] `package.json` 中的 `build` 命令配置正确
- [ ] 所有依赖在 `dependencies` 中（不要放在 `devDependencies`）
- [ ] 环境变量已在代码中使用 `import.meta.env` 正确获取
- [ ] 没有硬编码的本地地址（如 `localhost:3000`）
- [ ] `.gitignore` 配置正确（`node_modules/`、`.env` 等已忽略）

### 1.3 本地构建测试

在部署前，先在本地测试构建是否正常：

```bash
# 进入前端项目目录
cd apps/ai-lowcode

# 安装依赖
pnpm install

# 执行构建
pnpm build

# 预览构建结果
pnpm preview
```

确保构建过程没有错误，并且预览可以正常访问。

## 2. Vercel 部署步骤

### 2.1 方式一：通过 Vercel 控制台部署（推荐）

#### 步骤 1：登录 Vercel

访问 [https://vercel.com](https://vercel.com)，使用 GitHub 账号登录。

#### 步骤 2：导入项目

1. 点击 **Add New** → **Project**
2. 在 **Import Git Repository** 中选择你的仓库
3. 点击 **Import**

#### 步骤 3：配置项目

在项目配置页面填写以下信息：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **Project Name** | `ai-lowcode-platform` | 项目名称（可自定义） |
| **Framework Preset** | `Vite` | 自动检测或手动选择 |
| **Root Directory** | `apps/ai-lowcode` | 前端代码所在目录 |
| **Build Command** | `pnpm build` | 或留空使用默认值 |
| **Output Directory** | `dist` | Vite 默认输出目录 |
| **Install Command** | `pnpm install` | 或留空使用默认值 |

#### 步骤 4：配置环境变量

在 **Environment Variables** 部分添加：

```
VITE_API_URL=https://your-railway-app.railway.app
```

（将值替换为实际的后端 API 地址）

#### 步骤 5：部署

点击 **Deploy** 按钮，等待 1-3 分钟完成部署。

部署成功后，Vercel 会自动生成一个类似 `https://ai-lowcode-platform-xxx.vercel.app` 的域名。

---

### 2.2 方式二：通过 Vercel CLI 部署

#### 步骤 1：安装 Vercel CLI

```bash
npm install -g vercel
# 或使用 pnpm
pnpm add -g vercel
```

#### 步骤 2：登录 Vercel

```bash
vercel login
```

按照提示完成登录。

#### 步骤 3：执行部署

```bash
cd apps/ai-lowcode

# 预览部署（测试环境）
vercel

# 生产环境部署
vercel --prod
```

按照提示选择或确认以下信息：
- **Set up and deploy**：选择 `Yes`
- **Link to existing project**：选择 `No`（创建新项目）
- **In which directory is your code located**：选择当前目录 `.`
- **Project name**：输入项目名称
- **In which directory is your code located**：确认是 `apps/ai-lowcode`
- **Want to modify these settings**：选择 `No`
- **Want to link the project**：选择 `Yes`

---

### 2.3 方式三：通过 Git 自动部署

Vercel 可以与 Git 集成，实现：
- 推送到 `main`/`master` 分支 → 自动部署生产环境
- 创建 PR → 自动部署预览环境
- 合并 PR → 自动部署生产环境

**配置步骤**：
1. 在 Vercel 项目的 **Settings** → **Git** 中
2. 选择你的 Git 仓库
3. 配置分支和部署设置

## 3. 环境变量配置

### 3.1 必需环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `VITE_API_URL` | 后端 API 地址 | `https://ai-lowcode-api.railway.app` |

### 3.2 在 Vercel 控制台配置

1. 进入项目页面
2. 点击 **Settings** → **Environment Variables**
3. 点击 **Add New**
4. 填写变量名和值
5. 选择适用的环境（Development、Preview、Production）
6. 点击 **Save**

### 3.3 在项目代码中使用

使用 `import.meta.env` 访问环境变量：

```typescript
// lib/axios.ts
const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_URL || "http://localhost:3000/api",
});
```

**注意**：
- Vite 环境变量必须以 `VITE_` 开头才会暴露给客户端
- 不要在代码中提交敏感信息（如 API Key）

## 4. 构建配置

### 4.1 vercel.json 配置文件

在 `apps/ai-lowcode/` 目录下创建 `vercel.json` 文件：

```json
{
  "framework": "vite",
  "buildCommand": "pnpm build",
  "outputDirectory": "dist",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "X-Content-Type-Options",
          "value": "nosniff"
        },
        {
          "key": "X-Frame-Options",
          "value": "DENY"
        },
        {
          "key": "X-XSS-Protection",
          "value": "1; mode=block"
        }
      ]
    }
  ]
}
```

#### 配置说明：

| 配置项 | 说明 |
|--------|------|
| `rewrites` | 路由重写规则，解决 SPA 刷新 404 问题 |
| `headers` | 添加安全响应头 |

### 4.2 Vite 配置检查

确保 `vite.config.ts` 配置正确：

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import path from "path";

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
  server: {
    port: 5173,
    proxy: {
      "/api": {
        target: "http://localhost:3000",
        changeOrigin: true,
      },
    },
  },
});
```

## 5. 自定义域名（可选）

### 5.1 添加自定义域名

1. 进入 Vercel 项目的 **Settings** → **Domains**
2. 点击 **Add**，输入你的域名（如 `ai-lowcode.yourdomain.com`）
3. 点击 **Add**

### 5.2 配置 DNS

Vercel 会提供 DNS 配置指引，有两种方式：

#### 方式 A：使用 Vercel Nameservers（推荐）

在你的域名注册商处，将 Nameservers 更改为 Vercel 提供的 Nameservers：
- `ns1.vercel-dns.com`
- `ns2.vercel-dns.com`

#### 方式 B：添加 A 记录和 CNAME 记录

在你的 DNS 管理面板添加以下记录：

| 类型 | 主机记录 | 记录值 | TTL |
|------|----------|--------|-----|
| A | `@` | `76.76.21.21` | 自动 |
| CNAME | `www` | `cname.vercel-dns.com` | 自动 |

或针对子域名：

| 类型 | 主机记录 | 记录值 | TTL |
|------|----------|--------|-----|
| CNAME | `ai-lowcode` | `cname.vercel-dns.com` | 自动 |

### 5.3 等待 SSL 证书

Vercel 会自动为你的域名申请和配置 Let's Encrypt SSL 证书，通常几分钟内完成。

配置完成后，你的网站就可以通过 `https://ai-lowcode.yourdomain.com` 访问了。

## 6. 部署后验证

### 6.1 基础检查清单

部署完成后，请进行以下检查：

- [ ] 可以通过 Vercel 提供的域名正常访问网站
- [ ] 页面加载正常，没有空白或错误
- [ ] 浏览器控制台没有报错（按 F12 查看）
- [ ] 路由可以正常跳转
- [ ] 刷新页面不会出现 404（需要 vercel.json 配置）
- [ ] API 请求可以正常发送（Network 标签页检查）
- [ ] 登录/注册功能正常
- [ ] 可以创建应用、编辑工作流

### 6.2 浏览器控制台检查

1. 打开浏览器开发者工具（按 F12）
2. 切换到 **Console** 标签
3. 检查是否有 JavaScript 错误
4. 检查是否有 `Failed to fetch` 或网络错误

### 6.3 Network 检查

1. 打开浏览器开发者工具
2. 切换到 **Network** 标签
3. 刷新页面
4. 检查所有请求是否成功（状态码 200）
5. 检查 API 请求的地址是否正确（不是 localhost）

## 7. 常见问题排查

### 7.1 问题：构建失败

**可能原因**：
- Node.js 版本不兼容
- 依赖安装失败
- TypeScript 类型错误

**排查步骤**：
1. 查看 Vercel 构建日志
2. 在本地运行 `pnpm build` 检查是否有错误
3. 检查 `package.json` 中的 Node 版本要求
4. 检查 `node_modules/` 是否在 `.gitignore` 中

**解决方案**：
- 在项目根目录添加 `.nvmrc` 文件，指定 Node 版本：
  ```
  20
  ```
- 在 `package.json` 中添加 `engines` 字段：
  ```json
  {
    "engines": {
      "node": ">=18"
    }
  }
  ```

---

### 7.2 问题：部署后页面空白

**可能原因**：
- 环境变量未配置
- API 地址错误
- 路由配置问题

**排查步骤**：
1. 检查浏览器控制台是否有错误
2. 检查 `VITE_API_URL` 是否正确配置
3. 检查 Network 请求是否正常

---

### 7.3 问题：刷新页面出现 404

**可能原因**：
- 没有配置路由重写规则

**解决方案**：
添加 `vercel.json` 配置文件（见 4.1 节）

---

### 7.4 问题：API 请求失败（CORS 错误）

**可能原因**：
- 后端 CORS 配置不正确
- 前端 API 地址错误

**解决方案**：
- 检查后端 CORS 配置（见服务端部署文档）
- 确认 `VITE_API_URL` 配置正确

---

### 7.5 问题：环境变量未生效

**可能原因**：
- 环境变量没有以 `VITE_` 开头
- 修改环境变量后没有重新部署

**解决方案**：
- 确保环境变量名以 `VITE_` 开头
- 修改环境变量后，需要重新触发部署（可以重新推送代码或在 Vercel 控制台点击 Redeploy）

## 8. 最佳实践

### 8.1 环境管理

- **开发环境**：本地 `.env` 文件
- **预览环境**：Vercel Preview 环境变量
- **生产环境**：Vercel Production 环境变量

不要在代码中提交 `.env` 文件，应该提交 `.env.example` 作为模板：

```env
# .env.example
VITE_API_URL=http://localhost:3000/api
```

### 8.2 部署流程建议

1. **开发**：在本地开发，使用 `main`/`master` 分支
2. **功能开发**：创建 feature 分支，推送到 GitHub
3. **预览**：创建 PR，Vercel 自动部署预览环境
4. **测试**：在预览环境测试功能
5. **上线**：合并 PR 到 `main`/`master`，自动部署生产环境

### 8.3 性能优化

1. **启用 Vercel Edge Functions**（如果需要）
2. **使用 CDN 加速**：Vercel 默认使用 CDN
3. **图片优化**：使用 WebP 格式，适当压缩
4. **代码分割**：路由级别懒加载

### 8.4 监控和日志

- **Vercel Analytics**：查看访问数据和性能指标
- **Runtime Logs**：在 Vercel 项目的 **Logs** 中查看运行时日志
- **错误监控**：可以集成 Sentry 等监控服务

### 8.5 安全性建议

1. 不要在前端代码中暴露敏感信息
2. 使用 HTTPS（Vercel 自动提供）
3. 配置安全响应头（见 4.1 节）
4. 启用 XSS 防护
