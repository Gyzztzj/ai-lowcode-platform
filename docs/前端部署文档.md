# AI 低代码平台 - 前端部署文档

本指南将帮助您将 AI 低代码平台前端部署到 Vercel 免费平台。

## 前置条件

1. 已注册 Vercel 账号（推荐使用 GitHub 账号关联）
2. 项目代码已推送到 GitHub/GitLab/Bitbucket 仓库
3. 已完成服务端部署并获取服务端地址

## 部署步骤

### 1. Vercel 账号准备

1. 访问 [Vercel 官网](https://vercel.com)
2. 点击 Sign Up 注册账号，或点击 Login 登录
3. 推荐使用 GitHub 账号关联，方便后续项目导入
4. 完成账号基本设置（邮箱验证等）

### 2. 导入项目

1. 登录 Vercel 控制台
2. 点击 Dashboard 右上角的 Add New → Project
3. 在 Import Git Repository 部分选择您的仓库
4. 点击 Import 导入项目

### 3. 项目配置

在 Configure Project 页面进行以下配置：

#### 3.1 项目名称

- Project Name: 输入项目名称，或保持默认

#### 3.2 Root Directory

- Root Directory: 设置为 `web`（前端代码所在目录）

#### 3.3 框架预设

Vercel 会自动检测到这是一个 Vite 项目，Framework Preset 会自动选择 Vite，无需更改。

#### 3.4 构建命令

- Build Command: `pnpm build`
- Output Directory: `dist`

### 4. 环境变量配置

在 Environment Variables 部分添加以下环境变量：

| 变量名 | 值 | 必填 | 说明 |
|--------|-----|------|------|
| VITE_API_URL | `https://your-railway-api.railway.app` | ✅ | 服务端 API 地址 |

注意：将 `https://your-railway-api.railway.app` 替换为您实际部署的服务端地址。

### 5. 开始部署

1. 确认所有配置无误
2. 点击 Deploy 按钮开始部署
3. 等待构建完成（通常 1-3 分钟）
4. 部署成功后，Vercel 会分配一个类似 `https://your-project-name.vercel.app` 的域名

## 部署后验证

### 1. 验证前端访问

1. 在浏览器中访问 Vercel 分配的域名
2. 验证页面正常加载
3. 检查控制台是否有错误信息

### 2. 验证 API 连接

1. 测试登录功能，确认 API 请求正常
2. 验证其他功能是否工作正常

## 高级配置

### 1. 自定义域名

1. 进入项目 Settings → Domains
2. 输入您的自定义域名（如 `app.yourdomain.com`）
3. 点击 Add 按钮
4. 按照提示配置 DNS 记录

### 2. Vercel 配置文件

在项目根目录（不是 web 目录）创建 `vercel.json` 文件，配置重定向规则：

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

这可以解决路由刷新返回 404 的问题。

### 3. 构建命令优化

如果需要，可以在 web 目录的 package.json 中自定义构建命令：

```json
{
  "scripts": {
    "build": "vite build"
  }
}
```

## 常见问题

### 1. 构建失败，提示依赖安装错误

**问题**：构建过程中出现 pnpm install 相关错误

**解决方案**：
- 检查 `web/package.json` 中的依赖版本
- 确认 Node 版本兼容（推荐 Node 20+）
- 在本地先执行 `pnpm install` 和 `pnpm build`，确认没有问题

### 2. API 请求失败

**问题**：页面可以加载，但 API 请求失败

**解决方案**：
- 检查 `VITE_API_URL` 是否配置正确
- 确保服务端已部署并正常运行
- 检查服务端的 CORS 配置，允许 Vercel 域名访问

### 3. 路由刷新 404

**问题**：进入某个页面后刷新，返回 404 错误

**解决方案**：
- 在项目根目录创建 `vercel.json` 文件
- 添加重写规则，将所有请求重定向到 index.html

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

### 4. 环境变量不生效

**问题**：修改环境变量后没有生效

**解决方案**：
- 修改环境变量后需要重新部署才能生效
- 在 Vercel 控制台的项目设置中修改，不要在代码中硬编码

## 本地开发部署验证

在部署到 Vercel 之前，建议先在本地验证：

1. 确保前端可以正常构建：
```bash
cd web
pnpm install
pnpm build
```

2. 预览生产构建：
```bash
pnpm preview
```

3. 确认功能正常后再部署到 Vercel

## CI/CD 自动部署

Vercel 默认启用自动部署：

- 当推送到主分支（如 main/master）时，自动部署生产环境
- 当创建 Pull Request 时，自动部署预览环境
- 预览环境会有一个独立的 URL，可以用来验证变更

可以在项目 Settings → Git 中修改这些配置。

## 性能优化建议

### 1. 图片优化

- 使用 WebP 格式的图片
- 图片尺寸适配显示大小
- 使用 CDN 加速图片加载

### 2. 代码分割

Vite 会自动进行代码分割，路由级别的懒加载已内置。

### 3. 使用 Edge Functions

对于需要服务端处理的功能，可以使用 Vercel Edge Functions。

## 备份和回滚

### 1. 代码备份

代码已保存在 Git 仓库中，本身就是备份。

### 2. 回滚到之前版本

1. 进入项目 Deployments 页面
2. 找到需要回滚的部署
3. 点击该部署右侧的 ... 菜单
4. 选择 Promote to Production
