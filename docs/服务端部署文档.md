# AI 低代码平台 - 服务端部署文档

本指南将帮助您将 AI 低代码平台服务端部署到 Railway 免费平台。

## 前置条件

1. 已注册 Railway 账号（推荐使用 GitHub 账号关联）
2. 项目代码已推送到 GitHub/GitLab/Bitbucket 仓库
3. 已准备好 AI 模型 API 密钥（OpenAI/豆包等）

## 部署步骤

### 1. Railway 账号准备

1. 访问 [Railway 官网](https://railway.app)
2. 点击 Start New Project 或 Login 登录
3. 推荐使用 GitHub 账号关联，方便后续项目导入
4. 完成账号基本设置

### 2. 创建项目

#### 2.1 创建空项目

1. 登录 Railway 控制台
2. 点击 New Project
3. 选择 Empty Project
4. 为项目命名，如 "ai-lowcode-platform"

#### 2.2 或从 Git 仓库导入

1. 点击 New Project → Deploy from repo
2. 选择 GitHub 仓库中的 ai-lowcode-platform 项目
3. 点击 Deploy Now 继续

### 3. 添加数据库服务

#### 3.1 创建 PostgreSQL 数据库

1. 在项目页面点击 New → Database
2. 选择 PostgreSQL
3. 等待数据库创建完成（约 1-2 分钟）

#### 3.2 配置 pgvector 扩展

项目需要 pgvector 扩展支持向量存储。Railway 的 PostgreSQL 镜像默认已包含 pgvector，但建议确认。

如需要手动启用 pgvector 扩展，可以按以下步骤：

1. 进入数据库服务页面
2. 点击 Connect → Psql
3. 连接数据库后执行：
```sql
CREATE EXTENSION IF NOT EXISTS "vector";
```

数据库连接信息可以在 Variables 页面找到。

### 4. 添加 Redis 服务（可选但推荐）

1. 在项目页面点击 New → Database
2. 选择 Redis
3. 等待创建完成

Redis 用于：
- 请求限流
- 缓存
- Session 管理

### 5. 部署应用服务

#### 5.1 从 Git 仓库部署

1. 在项目页面点击 New → Web Service
2. 选择连接 GitHub 账户（如未连接）
3. 选择 ai-lowcode-platform 仓库
4. 配置部署参数

#### 5.2 Root Directory

- Root Directory: 设置为 `server`（服务端代码所在目录）

#### 5.3 构建和启动命令

在 Build 部分配置：

- Build Command: `pnpm install && pnpm build`
- Start Command: `pnpm start:prod`

### 6. 环境变量配置

在服务设置的 Variables 部分添加以下环境变量：

| 变量名 | 值 | 必填 | 说明 |
|--------|-----|------|------|
| PORT | `3000` | ✅ | 服务端口 |
| NODE_ENV | `production` | ✅ | 环境标识 |
| DATABASE_URL | （自动生成） | ✅ | PostgreSQL 连接地址 |
| REDIS_URL | （自动生成） | 🔄 | Redis 连接地址 |
| JWT_SECRET | 自定义强密码 | ✅ | JWT 密钥 |
| ALLOWED_ORIGINS | `https://your-vercel-app.vercel.app` | ✅ | 允许的跨域源 |
| OPENAI_API_KEY | 您的 OpenAI API Key | 🔄 | OpenAI API 密钥 |
| DOUBAO_API_KEY | 您的豆包 API Key | 🔄 | 豆包 API 密钥 |

**注意**：
- DATABASE_URL 和 REDIS_URL 会在添加对应服务后自动生成，无需手动输入
- ALLOWED_ORIGINS 可以设置多个域名，用逗号分隔，如：`https://app1.com,https://app2.com`
- 至少需要配置一种 AI 模型 API（OPENAI_API_KEY 或 DOUBAO_API_KEY）
- JWT_SECRET 建议使用强密码，可以使用在线工具生成

### 7. 资源配置

在 Settings → Resources 部分配置：

- **Memory**: 默认 512MB，推荐 1GB 或更多
- **CPU**: 默认即可，按需调整
- **Disk**: 默认即可
- **Replicas**: 1（免费版限制）

### 8. 开始部署

1. 确认所有配置无误
2. 点击 Deploy 按钮开始部署
3. 等待构建和部署完成（通常 3-5 分钟）
4. 部署成功后，Railway 会分配一个类似 `https://your-service-name.railway.app` 的域名

## 部署后验证

### 1. 验证服务运行

1. 访问分配的域名根路径，应该可以看到服务响应
2. 访问 `/api` 路径确认服务正常
3. 检查日志页面（View Logs）确认没有错误

### 2. 验证 Swagger 文档

访问 `https://your-service-name.railway.app/api/docs` 查看 API 文档。

### 3. 验证数据库连接

1. 使用数据库连接信息
2. 使用数据库管理工具（如 DBeaver、pgAdmin）连接验证
3. 检查表是否正确创建
4. 验证 pgvector 扩展是否启用

### 4. 验证完整功能

完整测试流程：
1. 用户注册/登录
2. 创建应用
3. 配置工作流
4. 创建知识库并上传文档
5. 进行 AI 对话
6. 测试 OpenAI 兼容 API
7. 验证审计日志和 Token 用量统计

## 数据迁移

### 自动同步（开发环境）

在开发环境，可以在 `database.module.ts` 中配置 `synchronize: true`，TypeORM 会自动创建和更新数据库表结构。

**注意**：不要在生产环境启用 `synchronize`，这可能导致数据丢失。

### 手动迁移（生产环境）

如果需要执行手动数据库迁移，可以使用以下步骤：

1. 确保本地安装了项目依赖
2. 配置本地的 `.env` 文件指向生产数据库（注意备份）
3. 执行迁移：

```bash
cd server
pnpm migration:generate -n Init
pnpm migration:run
```

但通常 Railway 和 TypeORM 的自动同步足够简单项目使用。

## 备份策略

### 数据库备份

Railway 提供自动备份功能：

1. 进入数据库服务页面
2. 点击 Backups
3. 配置备份频率：建议每日备份
4. 保留最近 7-30 天的备份
5. 可以手动创建备份点

### 恢复备份

如需从备份恢复：

1. 进入 Backups 页面
2. 找到需要恢复的备份
3. 点击该备份右侧的 ... 菜单
4. 选择 Restore

## 安全配置

### 1. 限制数据库访问

Railway 默认只允许项目内访问数据库，无需额外配置。

### 2. 使用强密码

- 确保 JWT_SECRET 使用强密码
- 定期更换 API 密钥
- 不要将敏感信息提交到代码仓库

### 3. 环境变量安全

- 环境变量通过 Railway 控制台配置，不提交到 Git
- 敏感变量（如 API Key）可以加密存储
- 定期轮换 JWT_SECRET 和其他敏感配置

### 4. HTTPS

Railway 自动为分配的域名提供 HTTPS，无需额外配置。

## 常见问题

### 1. 构建失败

**问题**：构建过程中出现错误

**解决方案**：
- 检查 `server/package.json` 中的依赖版本
- 确认 Node 版本兼容（推荐 Node 20+）
- 查看构建日志，定位具体错误
- 本地先执行 `pnpm install` 和 `pnpm build`，确认没有问题

### 2. 数据库连接失败

**问题**：应用启动时无法连接数据库

**解决方案**：
- 检查 DATABASE_URL 是否正确
- 确认数据库服务已正常启动
- 检查数据库连接数是否超限
- 查看应用日志中的详细错误信息

### 3. API 请求失败

**问题**：前端请求时出现 CORS 错误

**解决方案**：
- 检查 ALLOWED_ORIGINS 是否配置正确
- 确认包含前端的域名（包括 https://）
- 多个域名用逗号分隔
- 注意不要有多余的空格

### 4. pgvector 扩展不存在

**问题**：提示 vector 类型不存在

**解决方案**：
- 确认使用的 PostgreSQL 镜像支持 pgvector
- 手动执行 CREATE EXTENSION IF NOT EXISTS "vector";
- 联系 Railway 支持确认镜像配置

### 5. AI 模型请求失败

**问题**：调用 AI 模型时出现错误

**解决方案**：
- 检查 API Key 是否配置正确
- 确认 API Key 有效且有额度
- 查看应用日志中的详细错误
- 检查网络连接是否正常

## 高级配置

### 1. 自定义域名

1. 进入应用服务 Settings → Domains
2. 点击 Add Custom Domain
3. 输入您的自定义域名（如 `api.yourdomain.com`）
4. 按照提示配置 DNS 记录
5. 等待 DNS 生效（可能需要几分钟到几小时）

### 2. 环境变量分组

可以在 Railway 控制台中为不同环境（测试、生产）创建不同的变量组。

### 3. 自动部署配置

Railway 默认自动部署：
- 当推送到主分支时自动部署
- 可以在 Settings → Deployments 中配置

### 4. 使用 Railway CLI

安装 Railway CLI：
```bash
npm install -g @railway/cli
```

使用 CLI 部署：
```bash
cd server
railway login
railway init
railway up
```

## 本地开发部署验证

在部署到 Railway 之前，建议先在本地验证：

1. 确保服务端可以正常构建：
```bash
cd server
pnpm install
pnpm build
```

2. 使用 Docker Compose 启动本地环境（推荐）：
```bash
cd server
docker-compose up -d
```

3. 本地验证功能正常后再部署到 Railway

## 成本说明

Railway 免费版提供：
- $5 免费额度/月

- 适合开发和小型项目

超出免费额度后按使用量计费。对于生产环境，建议评估使用量并选择合适的付费方案。

## 扩展到其他平台

除了 Railway，服务端也可以部署到以下平台：

### 1. Render

- 类似 Railway，也提供免费额度
- 支持 PostgreSQL 和 Redis
- 部署方式与 Railway 类似

### 2. Fly.io

- 提供全球部署
- 支持 Docker 部署
- 配置稍复杂但更灵活

### 3. Heroku（已不再免费）

- 经典的 PaaS 平台
- 需要付费使用

### 4. 传统 VPS

- 可以部署到 AWS EC2、阿里云、腾讯云等
- 需要更多配置，但完全可控
- 需要自己配置数据库和 Redis
