# 服务端部署到 Railway - 超详细手把手教程

## 准备工作

1. 一个 GitHub 账号
2. 你的代码已经推送到 GitHub 上的 `ai-lowcode-platform` 仓库
3. 一个 Railway 账号（用 GitHub 登录就行）
4. 一个 OpenAI API Key 或者豆包 API Key（至少要有一个）

---

## 开始部署（每一步都跟着做）

---

### 第 1 步：打开 Railway 并登录

1. 打开浏览器，访问：https://railway.app
2. 点击右上角的 **Login** 按钮
3. 选择 **Sign in with GitHub**
4. 授权 Railway 访问你的 GitHub 账号（点同意/Authorize）
5. 登录成功后，你会看到 Railway 的控制台（Dashboard）

---

### 第 2 步：从 GitHub 仓库创建项目（关键！）

1. 在控制台页面，点击中间或右上角的 **New Project** 按钮
2. 你会看到几个选项，**选择 GitHub repo**（不是 Empty Project！）
3. 如果是第一次用，会提示你连接 GitHub 账号，按照提示授权
4. 授权后，在搜索框里输入你的仓库名：`ai-lowcode-platform`
5. 点击搜索结果里的你的仓库
6. 接下来你会看到两个按钮：
   - **Deploy Now**（直接部署）
   - **Add variables**（先配置）
     → **选择 Add variables**（先配置，不直接部署！）

---

### 第 3 步：配置服务的基本设置

现在你会进入服务的配置页面：

1. 找到 **Root Directory** 这一项（根目录）
   → 输入：`server`（很重要！因为服务端代码在 server 文件夹里）

2. 找到 **Branch** 这一项（分支）
   → 选择你的主分支，通常是 `main` 或 `master`

3. **Build Command**（构建命令）
   → 可以留空，或者输入：`pnpm install && pnpm build`

4. **Start Command**（启动命令）
   → 可以留空，或者输入：`pnpm start:prod`

5. 先别急着部署！我们还需要加数据库。

---

### 第 4 步：添加 PostgreSQL 数据库

1. 看页面顶部或左侧，找到你的项目画布（Project Canvas）
2. 在画布上，点击 **New** 按钮（在右上角）
3. 选择 **Database**
4. 选择 **PostgreSQL**
5. 等待 1-2 分钟，数据库会创建完成（你会看到状态变成 Ready）

---

### 第 5 步：启用 pgvector 扩展（重要！不然向量功能用不了）

1. 在项目画布上，点击刚创建好的 **PostgreSQL** 数据库（那个方块）
2. 会弹出一个面板，找到 **Connect** 标签页（或按钮）
3. 点击 **Connect** 后，选择 **Psql**（这会打开一个网页版的终端）
4. 在终端里，复制并粘贴下面这句话：
   ```sql
   CREATE EXTENSION IF NOT EXISTS "vector";
   ```
5. 按回车键执行
6. 如果看到类似 "CREATE EXTENSION" 的提示，说明成功了
7. 可以关掉这个终端页面了

---

### 第 6 步：添加 Redis（可选，但推荐加上）

1. 在项目画布上，再次点击 **New** 按钮
2. 选择 **Database**
3. 选择 **Redis**
4. 等待创建完成

---

### 第 7 步：配置环境变量（最容易出错的一步！）

回到你的服务（不是数据库！是那个从 GitHub 导入的服务方块）：

1. 在项目画布上，点击你的服务方块
2. 在弹出的面板里，点击 **Variables** 标签
3. 点击 **New Variable** 按钮，逐个添加下面的变量：

#### 7.1 必须手动添加的变量：

| 变量名                    | 要填的值               | 说明                                                                |
| ------------------------- | ---------------------- | ------------------------------------------------------------------- |
| `NODE_ENV`                | `production`           | 固定写这个                                                          |
| `JWT_SECRET`              | `随便写一串复杂的密码` | 比如：`my-super-secret-key-123-abc-xyz`                             |
| `ALLOWED_ORIGINS`         | `你的前端地址`         | 比如：`https://your-frontend.vercel.app` 或 `http://localhost:5173` |
| `OPENAI_API_KEY`          | `你的 OpenAI Key`      | 如果你用 OpenAI                                                     |
| `DOUBAO_API_KEY`          | `你的豆包 Key`         | 如果你用豆包                                                        |
| `DASHSCOPE_CHAT_API_KEY` | `你的通义千问 Key`     | 用于 rerank 功能需要（推荐添加）                                        |
| `RERANK_DEFAULT_MODEL`   | `qwen3-rerank`       | 默认 rerank 模型（可选，不填默认 qwen3-rerank）                         |

**注意**：OPENAI_API_KEY 和 DOUBAO_API_KEY 至少要填一个！如果需要使用 rerank 功能，必须配置 DASHSCOPE_CHAT_API_KEY！

#### 7.2 自动引用数据库变量（不要手动输入！）

1. 点击 **New Variable**
2. 找到 **Add Reference** 或 **Reference Variable** 选项（可能是一个链接或按钮）
3. 选择 **PostgreSQL**，然后选择 `DATABASE_URL`
4. 点击 **Add** 或 **Add Variable**
5. 如果你添加了 Redis，同样操作：
   - 点击 **New Variable** → **Add Reference**
   - 选择 **Redis**，选择 `REDIS_URL`
   - 点击 **Add**

---

### 第 8 步：终于可以部署了！

1. 检查一下所有配置都好了吗：
   - Root Directory 是 `server`
   - 环境变量都加了
   - 数据库和 Redis 都创建了

2. 点击页面顶部的 **Deploy** 或 **Deploy Now** 按钮
3. 等待 3-5 分钟（期间可以在 **Deployments** 标签页看进度）
4. 部署成功后，你会看到一个网址，类似：`https://ai-lowcode-platform-production.up.railway.app`

---

### 第 9 步：验证部署成功

1. 复制那个 Railway 分配给你的网址
2. 在浏览器里打开它
3. 如果自动跳转到了 `/api/docs` 页面（API 文档页面），说明部署成功！
4. 你也可以访问 `https://你的网址/api` 看看响应

---

## 常见问题（按可能性排序）

### Q1: 构建失败了怎么办？

A:

1. 点击你的服务 → **Deployments** 标签页
2. 点击最近一次失败的部署，看日志
3. 常见原因：
   - 忘记设置 Root Directory 为 `server`
   - 本地先试一下：`cd server && pnpm install && pnpm build`，看本地能不能成功

### Q2: 数据库连不上？

A:

1. 检查 DATABASE_URL 是不是通过 Add Reference 添加的
2. 不要手动输入 DATABASE_URL！一定要用 Reference
3. 检查 PostgreSQL 数据库状态是不是 Ready

### Q3: 前端访问显示 CORS 错误？

A:

1. 检查 ALLOWED_ORIGINS 变量
2. 确保包含了前端的完整地址，包括 `https://` 或 `http://`
3. 多个地址用逗号分隔，不要加空格

### Q4: pgvector 相关的错误？

A:

1. 再检查一下第 5 步做了没
2. 再执行一次：`CREATE EXTENSION IF NOT EXISTS "vector";`

### Q5: Root Directory 找不到在哪里设置？

A:

1. 点击你的服务方块
2. 点击 **Settings** 标签
3. 往下翻，找到 **Root Directory** 或 **Service Source** 部分

## 本地测试（部署前先试试，避免浪费时间）

如果你想在本地先测试一下服务端能不能跑：

```bash
# 进入 server 目录
cd server

# 安装依赖
pnpm install

# 构建
pnpm build

# （可选）用 Docker Compose 启动本地数据库和服务
docker-compose up -d
```

---

## 部署成功后要做的

1. 把 Railway 分配给你的网址发给前端同学
2. 让前端同学把他们的请求地址改成这个网址
3. 测试一下完整功能（注册、登录、创建应用、知识库等）
